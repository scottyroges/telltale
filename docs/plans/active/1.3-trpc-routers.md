# Plan 1.3: tRPC Routers

**Status:** Planned
**Depends on:** 1.2 (needs conversation service)
**Blocks:** 1.4

## Goal

Create tRPC routers for story CRUD, session management, and sending messages. This completes the backend API surface for the conversation engine ‚Äî all through tRPC with full type safety.

## Manual Steps (Before Code)

None.

## Key Steps

- Create `src/server/routers/story.ts`:
  - `create` ‚Äî protected mutation. Input: `{ title: string }`. Creates a Book for the user (if none exists) or uses their existing book, then creates a Story. Returns the story.
  - `list` ‚Äî protected query. Returns all stories for the user's book(s), including session count and status.
  - `getById` ‚Äî protected query. Input: `{ id: string }`. Returns story with sessions. Verifies the story belongs to the user.
- Create `src/server/routers/session.ts`:
  - `start` ‚Äî protected mutation. Input: `{ storyId: string }`. Delegates to `conversationService.startSession()`. Returns `{ sessionId, openingMessage }`. Verifies the story belongs to the user.
  - `sendMessage` ‚Äî protected mutation. Input: `{ sessionId: string, content: string }`. Delegates to `conversationService.sendMessage()`. Returns the assistant's response text. Verifies the session belongs to the user.
  - `getMessages` ‚Äî protected query. Input: `{ sessionId: string }`. Delegates to `conversationService.getSessionMessages()`. Verifies the session belongs to the user.
  - `complete` ‚Äî protected mutation. Input: `{ sessionId: string }`. Delegates to `conversationService.completeSession()`. Verifies the session belongs to the user.
- Update `src/server/routers/_app.ts`:
  - Add `story: storyRouter` and `session: sessionRouter` to the root router
- Create ownership verification helper:
  - `src/server/routers/_helpers.ts` or similar ‚Äî a shared function to verify resource ownership (session ‚Üí story ‚Üí book ‚Üí userId chain)
  - Used by tRPC routers to enforce access control
- Write tests:
  - `tests/server/routers/story.test.ts` ‚Äî test create, list, getById with mock repositories. Test UNAUTHORIZED without auth. Test that stories from other users are not accessible.
  - `tests/server/routers/session.test.ts` ‚Äî test start, sendMessage, getMessages, complete with mock service. Test ownership verification.

## Notes

- The `story.create` router handles the "first-time user" case: if the user has no Book yet, it creates one with a default title (e.g., "My Life Story"). This simplifies the MVP ‚Äî users don't manage Books directly.
- Per ADR 013, all conversation operations go through tRPC ‚Äî no separate streaming API route. This gives us full type safety and a single API surface with no duplicated auth.
- The `sendMessage` mutation awaits the full response from the conversation service and returns it. The client sees a loading state while waiting, then gets the complete response. For short interviewer responses this is snappy.
- Ownership verification is important: a user should never be able to access another user's sessions, stories, or messages. The chain is always: resource ‚Üí ... ‚Üí book ‚Üí check `book.userId === ctx.userId`.
- Middleware (`src/middleware.ts`) already allows `/api/*` as public routes (auth is enforced at the handler level), so no middleware changes needed.
