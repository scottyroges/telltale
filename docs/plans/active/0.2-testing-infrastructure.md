# Plan 0.2: Testing Infrastructure

**Status:** Not Started
**Depends on:** 0.1 (need project structure and `package.json`)
**Blocks:** None (but all subsequent plans should include tests from the start)

## Goal

Install and configure Vitest, React Testing Library, and Playwright per ADR 010. Establish the hybrid test file organization (co-located for frontend, mirrored `tests/` for backend), create test helpers, and add a smoke test for each runner to prove the setup works.

## Manual Steps (Before Code)

None.

## Key Steps

### Vitest + React Testing Library

- Install `vitest`, `@vitejs/plugin-react`, `vite-tsconfig-paths`, `jsdom`
- Install `@testing-library/react`, `@testing-library/jest-dom`, `@testing-library/user-event`
- Create `vitest.config.ts`:
  - `plugins: [react(), tsconfigPaths()]`
  - `test.environment: "jsdom"` for component tests
  - `test.include` covering both `src/**/*.test.{ts,tsx}` and `tests/**/*.test.{ts,tsx}`
  - `test.setupFiles` pointing to `src/test/setup.ts`
- Create `src/test/setup.ts` ‚Äî imports `@testing-library/jest-dom/vitest`
- Create `tests/helpers/` directory for shared test utilities (factories, mocks)
- Add `package.json` scripts:
  - `"test": "vitest run"`
  - `"test:watch": "vitest"`
  - `"test:coverage": "vitest run --coverage"`

### Playwright

- Install `@playwright/test`
- Run `npx playwright install --with-deps chromium` (Chromium only for dev, full matrix in CI)
- Create `playwright.config.ts`:
  - `testDir: "./e2e"`
  - `webServer` config to start Next.js on a test port
  - Projects: Chromium only for local dev, add Firefox + WebKit in CI
  - `baseURL: "http://localhost:3000"`
- Add `package.json` scripts:
  - `"test:e2e": "playwright test"`
  - `"test:e2e:ui": "playwright test --ui"`

### Smoke Tests

- Create `tests/smoke.test.ts` ‚Äî trivial assertion to prove Vitest runs for backend tests
- Create `src/components/home-page.test.tsx` ‚Äî render test for an existing component to prove RTL works with co-located frontend tests
- Create `e2e/smoke.spec.ts` ‚Äî navigate to `/` and assert the page loads

### Configuration

- Add to `.gitignore`: `playwright-report/`, `test-results/`, `coverage/`
- Update `tsconfig.json` if needed to include `tests/` in compilation

## Notes

- Vitest config must resolve the `@/*` path alias via `vite-tsconfig-paths` ‚Äî without this, imports break in test files
- `jsdom` environment is needed for component tests but not for pure logic tests ‚Äî Vitest uses the `environment` option or per-file `@vitest-environment` comments
- Playwright `webServer` should use `npm run dev` for local and `npm run build && npm start` for CI
- Only install Chromium locally ‚Äî the full browser matrix runs in CI to keep local setup fast

## Done When

- `npm test` runs Vitest and the smoke test passes
- `npm run test:watch` starts Vitest in watch mode
- `npm run test:e2e` starts Next.js, runs Playwright, and the smoke spec passes
- Co-located frontend test (`src/components/home-page.test.tsx`) is picked up by Vitest
- Mirrored backend test (`tests/smoke.test.ts`) is picked up by Vitest
- `npm run build` still succeeds (test files aren't included in the production build)
