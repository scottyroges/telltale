**Status:** Complete
**Created:** 2026-02-26
**Parent plan:** 2a (core memory block)
**Blocks:** 2a.2 (system prompt + response format)

## Goal

Add the `coreMemory` field to the Book model and expose it through the repository layer. This is pure data plumbing ‚Äî no behavior changes, no prompt changes, no conversation loop changes. After this PR, the field exists, can be read and written, and all existing tests pass with the new field.

## Changes

### 1. Prisma Schema

Add `coreMemory String?` to the Book model in `prisma/schema.prisma`:

```prisma
model Book {
  id            String         @id @default(cuid())
  userId        String
  title         String
  coreMemory    String?
  status        BookStatus     @default(IN_PROGRESS)
  // ... rest unchanged
}
```

Run migration: `npx prisma migrate dev --name add-core-memory-to-book`

Regenerate Kysely types: `npx prisma generate` (updates `src/db/types.ts` via prisma-kysely)

### 2. Domain Type

Add `coreMemory: string | null` to the `Book` type in `src/domain/book.ts`.

### 3. Book Repository

In `src/repositories/book.repository.ts`:

- Add `"coreMemory"` to the `columns` array so it's returned by queries that use `columns`
- Add `"book.coreMemory"` to the hardcoded select list in `findByIdWithDetails()` (this method doesn't use the shared `columns` array)
- Add new method:

```typescript
async updateCoreMemory(bookId: string, content: string): Promise<Book> {
  return db
    .updateTable("book")
    .set({ coreMemory: content, updatedAt: new Date() })
    .where("id", "=", bookId)
    .returning([...columns])
    .executeTakeFirstOrThrow();
},
```

### 5. New Tests
