# Plan 1.6: Context Window Management

**Status:** Planned
**Depends on:** 1.2 (needs conversation service + LLM provider), 1.5 (needs insight extraction for context injection)
**Blocks:** None in Phase 1

## Goal

Manage the context window for long conversations — summarize older messages into InterviewSummary records, keep recent ones verbatim, inject insight context for coherence. Without this, conversations longer than ~20 messages will hit the context limit or become expensive/slow due to sending the full history every turn.

## Manual Steps (Before Code)

None.

## Key Steps

- Create `src/services/summarization-prompt.ts`:
  - Exports `SUMMARIZATION_PROMPT` as a string constant
  - Instructs the LLM to summarize a conversation excerpt
  - Instructions: preserve names, dates, places, emotional moments, key narrative details. Be concise but don't lose specificity. Output a prose summary, not bullet points.
- Create `src/services/context.service.ts`:
  - Imports `llmProvider` from `@/lib/llm` (typed as `LLMProvider`)
  - **`buildContextWindow(interviewId: string)`**
    - Loads all messages for the interview via `messageRepository.findByInterviewId()`
    - Loads existing insights for the interview via `insightRepository.findByInterviewId()`
    - Loads the latest InterviewSummary via `interviewSummaryRepository.findLatestByInterviewId()`
    - Estimates total token count using a heuristic (chars / 4)
    - If total tokens are under the threshold (e.g., 8,000 tokens), return all messages as-is — no summarization needed
    - If total tokens exceed the threshold:
      - Split messages into "old" (to summarize) and "recent" (to keep verbatim, e.g., last 10 messages)
      - Check if the interview already has a stored summary (latest InterviewSummary)
      - If no stored summary, or if there are old messages beyond the summarized range (messageCount): call `summarizeMessages()` to create a new summary
      - Create a new InterviewSummary via `interviewSummaryRepository.create()` with `parentSummaryId` pointing to the previous summary (linked list pattern from 1.1)
    - Assemble and return the context window:
      1. System prompt (from `prompt.ts`)
      2. If summarized: a summary message framed as "Here is a summary of the earlier conversation: ..."
      3. Insight context: a message listing unexplored insights, framed as "Topics mentioned but not yet explored: ..."
      4. Recent messages verbatim
    - Returns `{ systemPrompt: string, messages: LLMMessage[] }` — ready to pass to `llmProvider.generateResponse()`
  - **`summarizeMessages(messages: Message[], existingSummary?: string)`**
    - Builds the input: if there's an existing summary, includes it as context ("Previous summary: ...") followed by the new messages to incorporate
    - Calls `llmProvider.generateResponse()` with the summarization prompt and message content
    - Returns the summary string
  - **Token counting heuristic:**
    - `estimateTokens(text: string): number` — `Math.ceil(text.length / 4)`
    - Simple but effective for English text. No need for a tokenizer library.
    - Constants: `MAX_CONTEXT_TOKENS = 16000`, `SUMMARIZATION_THRESHOLD = 8000`, `RECENT_MESSAGES_COUNT = 10`
- Update `src/services/conversation.service.ts`:
  - `sendMessage` now uses `contextService.buildContextWindow(interviewId)` instead of loading raw message history
  - The context window output goes directly to `llmProvider.generateResponse(systemPrompt, messages)`
  - The raw user message is still persisted as before — context window assembly doesn't affect persistence
- Write unit tests in `tests/services/context.service.test.ts`:
  - Mock `@/lib/llm` (the provider instance)
  - Mock all repositories
  - Use `// @vitest-environment node` directive
  - Test: under threshold — returns all messages, no summarization called
  - Test: over threshold — splits messages, calls summarization, creates InterviewSummary with correct parentSummaryId
  - Test: with existing summary — incorporates previous summary, only summarizes messages beyond messageCount
  - Test: insight context injection — includes unexplored insights in the context
  - Test: token estimation heuristic

## Notes

- The context window strategy is: **summarize old, keep recent, inject insights**. This is a common pattern for long-context chat applications.
- Summarization is **incremental** — we create new InterviewSummary records in a linked list (each pointing to its parent). Each summary incorporates the previous summary plus new messages. This avoids re-summarizing the entire conversation every turn.
- The InterviewSummary model from 1.1 tracks `messageCount` (cumulative messages covered) so the context service knows which messages are already summarized and only needs to incorporate new ones.
- The threshold constants (8K tokens for summarization trigger, 16K max context, 10 recent messages) are tuned conservatively for cost/latency management, not avoiding hard context limits.
- Insight context injection serves two purposes: (1) reminds the AI about topics to explore, improving conversation depth, and (2) preserves important details that might be lost during summarization.
- The summarization prompt tells the LLM to preserve specificity — "Maria, the sister who lived in Calabria" not "a family member". This is critical because the summary replaces the original messages.
- `buildContextWindow` returns `{ systemPrompt, messages: LLMMessage[] }` — provider-agnostic format ready for `llmProvider.generateResponse()`. No provider-specific types leak into the context service.
- If summarization fails (LLM error), fall back to truncating old messages rather than failing the conversation. Degraded but functional.
