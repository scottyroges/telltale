# Plan 2b.4: Onboarding UI

**Status:** Not Started
**Created:** 2026-03-02
**Parent plan:** 2b (onboarding + interview planning)
**Depends on:** 2b.3 (onboarding service + tRPC router)
**Blocks:** 2b.5 (plan / progress view)

## Goal

Add the onboarding page where users answer prompts about ~10 life areas, and a CTA on the interviews page that routes them there. The form auto-saves on blur, validates a minimum of 2 responses, and triggers plan generation with a loading state. After completion, the user is redirected back to the interviews page.

After this PR, a user can go through the full onboarding flow: interviews page → CTA → onboarding form → fill areas → generate plan → redirect back to interviews.

## Changes

### 1. Onboarding Page (Server Component)

**`src/app/(app)/book/[bookId]/onboarding/page.tsx`** — new file:

```typescript
import { redirect } from "next/navigation";
import { serverTRPC } from "@/lib/trpc/server";
import { LifeAreaForm } from "@/components/onboarding/life-area-form";
import styles from "./onboarding.module.css";

type OnboardingPageProps = {
  params: Promise<{ bookId: string }>;
};

export default async function OnboardingPage({ params }: OnboardingPageProps) {
  const { bookId } = await params;
  const trpc = await serverTRPC();
  const { onboardingStatus, responses } = await trpc.onboarding.getStatus({
    bookId,
  });

  if (onboardingStatus === "COMPLETE") {
    redirect(`/book/${bookId}/interviews`);
  }

  return (
    <div className={styles.page}>
      {/* Back link, heading, description */}
      <LifeAreaForm bookId={bookId} initialResponses={responses} />
    </div>
  );
}
```

Follows the existing server component pattern from the interviews page:
- `await params` (Next.js 15 async params)
- Single `await serverTRPC()` call, reuse for all queries
- Load `onboardingStatus` and `responses` via `onboarding.getStatus`
- Redirect to interviews if already `COMPLETE`
- Pass `initialResponses` to the client component for pre-filling (resume case)

**`src/app/(app)/book/[bookId]/onboarding/onboarding.module.css`** — new file. Basic page layout matching the interviews page pattern (`.page` with `max-width: 40rem`, heading styles, etc.)

### 2. Life Area Form (Client Component)

**`src/components/onboarding/life-area-form.tsx`** — new file:

```typescript
"use client";

import { useState, useCallback } from "react";
import { useRouter } from "next/navigation";
import { useMutation } from "@tanstack/react-query";
import { useTRPC } from "@/lib/trpc/client";
import { LIFE_AREAS } from "@/domain/life-area";
import type { LifeAreaResponse } from "@/domain/life-area";
import { LifeAreaCard } from "./life-area-card";
import styles from "./life-area-form.module.css";

type LifeAreaFormProps = {
  bookId: string;
  initialResponses: LifeAreaResponse[];
};
```

State and behavior:

- **Local state:** `Map<string, string>` of `lifeAreaKey → content`, initialized from `initialResponses`
- **Auto-save on blur:** When a textarea blurs with non-empty content that differs from the last-saved value, fire `onboarding.saveResponse` mutation. When blurring with empty content and a response previously existed, fire `onboarding.deleteResponse`.
- **Generate button:** "Generate My Interview Plan" button at the bottom. Enabled when `>= 2` areas have non-empty content. Disabled while any save is in flight or while the complete mutation is pending.
- **Complete mutation:** Fires `onboarding.complete` on click. Shows loading state ("Generating your interview plan..." or similar). On success, `router.push(`/book/${bookId}/interviews`)`.
- **Error handling:** Show error text below the generate button if the complete mutation fails.

Mutation setup (follows `useMutation` + `mutationOptions` pattern):

```typescript
const trpc = useTRPC();
const router = useRouter();

const saveResponse = useMutation(
  trpc.onboarding.saveResponse.mutationOptions(),
);

const deleteResponse = useMutation(
  trpc.onboarding.deleteResponse.mutationOptions(),
);

const complete = useMutation(
  trpc.onboarding.complete.mutationOptions({
    onSuccess() {
      router.push(`/book/${bookId}/interviews`);
    },
  }),
);
```

The `handleBlur` callback for each card:

```typescript
const handleBlur = useCallback(
  (lifeAreaKey: string, content: string) => {
    const trimmed = content.trim();
    const hadPreviousResponse = /* check initialResponses or saved state */;

    if (trimmed) {
      saveResponse.mutate({ bookId, lifeAreaKey, content: trimmed });
    } else if (hadPreviousResponse) {
      deleteResponse.mutate({ bookId, lifeAreaKey });
    }
  },
  [bookId, saveResponse, deleteResponse],
);
```

Render structure:

```tsx
<div className={styles.form}>
  <div className={styles.cards}>
    {LIFE_AREAS.map((area) => (
      <LifeAreaCard
        key={area.key}
        lifeArea={area}
        initialContent={responseMap.get(area.key) ?? ""}
        onBlur={handleBlur}
      />
    ))}
  </div>

  <div className={styles.footer}>
    <button
      className={styles.generateButton}
      disabled={filledCount < 2 || complete.isPending}
      onClick={() => complete.mutate({ bookId })}
    >
      {complete.isPending
        ? "Generating your interview plan..."
        : "Generate My Interview Plan"}
    </button>
    {complete.isError && (
      <p className={styles.error}>{complete.error.message}</p>
    )}
  </div>
</div>
```

**`src/components/onboarding/life-area-form.module.css`** — new file. Styles for the card list, footer, generate button, and error state.

Simple controlled textarea component:

- Textarea with local `content` state, initialized from `initialContent`
- Calls `onBlur(lifeArea.key, content)` when the textarea loses focus
- No save indicator on the card itself — auto-save is fire-and-forget

```tsx

  <h3 className={styles.name}>{lifeArea.name}</h3>
  <p className={styles.prompt}>{lifeArea.prompt}</p>
  <textarea
    className={styles.textarea}
    value={content}
    onChange={(e) => setContent(e.target.value)}
    onBlur={() => onBlur(lifeArea.key, content)}
    rows={4}
    placeholder="Share as much or as little as you'd like..."
  />
```
