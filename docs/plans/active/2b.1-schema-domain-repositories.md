# Plan 2b.1: Onboarding — Schema, Domain Types, and Repositories

**Status:** Not Started
**Created:** 2026-03-02
**Parent plan:** 2b (onboarding + interview planning)
**Blocks:** 2b.2 (plan generation service + prompt)

## Goal

Add all new database models (LifeAreaResponse, InterviewTopic, Thread), new enums (OnboardingStatus, TopicStatus, ThreadStatus), and additive fields on existing models (Book.onboardingStatus, Interview.topicId). Create the corresponding domain types, life area constants, and repository layer. Rename Question.category → lifeAreaKey and add Question.title so the catalog aligns with the life area taxonomy.

This is delivered as two PRs:

- **PR 1 (Phase 1a):** New models, enums, and additive fields. Purely additive — nothing breaks.
- **PR 2 (Phase 1b):** Rename Question.category → lifeAreaKey, add Question.title. Isolated because it's a disruptive rename touching many files.

### 1. Prisma Schema

Add three new enums to `prisma/schema.prisma`:

```prisma
enum OnboardingStatus {
  NOT_STARTED
  IN_PROGRESS
  COMPLETE
}

enum TopicStatus {
  SUGGESTED
  ACCEPTED
  COMPLETED
  SKIPPED
}

enum ThreadStatus {
  OPEN
  EXPLORED
  CLOSED
}
```

Add `onboardingStatus` field to the Book model, plus new relations:

```prisma
model Book {
  // ... existing fields ...
  onboardingStatus  OnboardingStatus @default(NOT_STARTED) @map("onboarding_status")
  lifeAreaResponses LifeAreaResponse[]
  interviewTopics   InterviewTopic[]
  threads           Thread[]
}
```

Add `topicId` field and relation to Interview model:

```prisma
model Interview {
  // ... existing fields ...
  topicId  String?         @map("topic_id")
  topic    InterviewTopic? @relation(fields: [topicId], references: [id])
  threads  Thread[]
}
```

Note: Interview already has `topic String` (a text field). The new `topicId` is a FK to InterviewTopic — different from the existing text field. The Prisma relation name `topic` conflicts with the existing `topic String` column. **Rename the relation field** to avoid collision — use `interviewTopic` for the relation:

```prisma
model Interview {
  // ... existing fields ...
  topicId          String?         @map("topic_id")
  interviewTopic   InterviewTopic? @relation(fields: [topicId], references: [id])
  threads          Thread[]
}
```

New `LifeAreaResponse` model:

```prisma
model LifeAreaResponse {
  id          String   @id @default(cuid())
  bookId      String   @map("book_id")
  lifeAreaKey String   @map("life_area_key")
  content     String
  book        Book     @relation(fields: [bookId], references: [id], onDelete: Cascade)
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")

  @@unique([bookId, lifeAreaKey])
  @@index([bookId])
  @@map("life_area_response")
}
```

New `InterviewTopic` model:

```prisma
model InterviewTopic {
  id                 String      @id @default(cuid())
  bookId             String      @map("book_id")
  title              String
  description        String
  primaryLifeAreaKey String      @map("primary_life_area_key")
  lifeAreaKeys       String[]    @map("life_area_keys")
  orderIndex         Int         @map("order_index")
  status             TopicStatus @default(SUGGESTED)
  book               Book        @relation(fields: [bookId], references: [id], onDelete: Cascade)
  interviews         Interview[]
  threads            Thread[]
  createdAt          DateTime    @default(now()) @map("created_at")
  updatedAt          DateTime    @updatedAt @map("updated_at")

  @@index([bookId])
  @@map("interview_topic")
}
```

New `Thread` model:

```prisma
model Thread {
  id          String          @id @default(cuid())
  bookId      String          @map("book_id")
  interviewId String          @map("interview_id")
  topicId     String?         @map("topic_id")
  title       String
  description String
  status      ThreadStatus    @default(OPEN)
  book        Book            @relation(fields: [bookId], references: [id], onDelete: Cascade)
  interview   Interview       @relation(fields: [interviewId], references: [id], onDelete: Cascade)
  topic       InterviewTopic? @relation(fields: [topicId], references: [id])
  createdAt   DateTime        @default(now()) @map("created_at")
  updatedAt   DateTime        @updatedAt @map("updated_at")

  @@index([bookId])
  @@index([interviewId])
  @@map("thread")
}
```

Run migration: `npx prisma migrate dev --name add-onboarding-models`

Regenerate Kysely types: `npx prisma generate` (updates `src/db/types.ts` and `src/db/enums.ts`)

### 3. Repository Layer

- Add new method:

```typescript
async updateOnboardingStatus(id: string, status: OnboardingStatus): Promise<Book> {
  return db
    .updateTable("book")
    .set({ onboardingStatus: status, updatedAt: new Date() })
    .where("id", "=", id)
    .returning([...columns])
    .executeTakeFirstOrThrow();
},
```

**`src/repositories/life-area-response.repository.ts`** — new file:

Methods:
- `upsert(bookId, lifeAreaKey, content)` — insert or update on `(bookId, lifeAreaKey)` unique constraint. Uses Kysely's `onConflict` with merge.

- `findById(id)` — returns `InterviewTopic | null`
- `updateStatus(id, status)` — updates status, returns `InterviewTopic`
