# Plan 0.7: CI Quality Gates

## Status: In Progress

## Goal

Add a GitHub Actions CI workflow as a quality gate on PRs and pushes to `main`.

## Decisions

- **E2E tests:** Deferred — they require a database + running server, disproportionate complexity for 2 spec files. Add later when the suite grows.
- **Build step:** Deferred — `npm run build` runs `prisma migrate deploy` which needs a real database. Vercel handles builds.
- **Type checking (`tsc --noEmit`):** Deferred — ESLint with `next/typescript` already catches type errors.
- **Single job:** Lint and tests share identical setup. A single job avoids ~30s of duplicated setup overhead.
- **No env vars needed:** `prisma generate` (via `postinstall`), ESLint, and all unit tests work without any secrets or environment variables.

## Files Created

- `.github/workflows/ci.yml` — CI workflow
- `docs/plans/active/0.7-ci-quality-gates.md` — this file

## Manual Step (After Merge)

Enable GitHub branch protection:
1. GitHub repo → Settings → Branches
2. Add rule for `main`
3. Enable "Require status checks to pass before merging"
4. Search for and add `Lint & Test` as a required check

## Verification

1. Push the workflow to `main`
2. Confirm it runs successfully in the Actions tab (lint + unit tests pass)
3. Create a test PR, confirm the `Lint & Test` check appears
4. Enable branch protection (manual step above)
