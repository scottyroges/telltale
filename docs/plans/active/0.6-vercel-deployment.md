# Plan 0.6: Vercel Deployment

**Status:** Not Started
**Depends on:** 0.5 (need the full working application)
**Blocks:** Nothing — this completes Phase 0

## Goal

Deploy to Vercel with environment variables, automatic deploys on push to `main`, preview deploys on PRs, and automated production database migrations.

## Manual Steps (Before Code)

1. **Create Vercel account** at https://vercel.com (free tier, use GitHub account)

2. **Install Vercel CLI:**
   ```
   npm install -g vercel
   ```

3. **Link the project:**
   ```
   vercel link
   ```
   - Create new project, name it "telltale"

4. **Set environment variables** in Vercel dashboard (all environments):
   - `DATABASE_URL` — Neon pooled connection string
   - `DIRECT_URL` — Neon direct connection string
   - `BETTER_AUTH_SECRET` — same as local
   - `BETTER_AUTH_URL` — production URL (e.g., `https://<your-domain>.vercel.app`)
   - `GOOGLE_CLIENT_ID` — Google OAuth client ID
   - `GOOGLE_CLIENT_SECRET` — Google OAuth client secret

5. **Update Google OAuth redirect URIs** in Google Cloud Console:
   - Add: `https://<your-domain>.vercel.app/api/auth/callback/google`
   - Optionally add preview deploy pattern

## Key Steps

- Update `package.json` build script: `"build": "prisma migrate deploy && next build"`
  - `prisma migrate deploy` applies pending migrations (idempotent, safe for every deploy)
- Update `.env.example` with all Phase 0 variables documented with comments
- Deploy via `vercel --prod` or push to `main`

## Notes

- Vercel auto-detects Next.js — no `vercel.json` needed
- `postinstall: "prisma generate"` (from Plan 0.3) runs during Vercel's `npm install` step
- `prisma migrate deploy` uses `DIRECT_URL` (bypasses pooler for DDL)
- Free tier: 100 GB bandwidth, 100 GB-hours serverless, 10s function timeout
- Preview deploys happen automatically on PRs
