## Key Steps

### Question Router (`src/server/routers/question.ts`)

| Operation | Type | Input | Delegates to | UI Usage |
|-----------|------|-------|-------------|----------|
| `list` | query | — | `questionRepository.findAll()` | Interviews page — browse the global question catalog |
| `listByCategory` | query | `{ category: string }` | `questionRepository.findByCategory()` | Interviews page — filter catalog by category |
| `create` | mutation | `{ category: string, prompt: string, orderIndex: number }` | `questionRepository.create()` | Admin — seed/manage the question catalog. No authorization gating yet. |

- All procedures are protected (require auth). No ownership checks — questions are global seed data.

### Book Router (`src/server/routers/book.ts`)

| Operation | Type | Input | Delegates to | UI Usage |
|-----------|------|-------|-------------|----------|
| `create` | mutation | `{ title: string }` | `bookRepository.create({ userId, title })` | Create a new book for the user. Title provided by client. |
| `getById` | query | `{ id: string }` | `bookRepository.findById()` | Load a book with its BookQuestions (including Question data and status) and Interviews. Used on the interviews page to show the checklist. |
| `list` | query | — | `bookRepository.findByUserId()` | List all books for the authenticated user. Used for a book picker / dashboard. |
| `addQuestion` | mutation | `{ bookId: string, questionId: string }` | `bookQuestionRepository.getNextOrderIndex()` then `bookQuestionRepository.create()` | Interviews page — add a question from the catalog to the book. Validates questionId exists via `questionRepository.findById()`, throws `NOT_FOUND` if missing. Gets orderIndex via `bookQuestionRepository.getNextOrderIndex(bookId)`. |
| `removeQuestion` | mutation | `{ bookQuestionId: string }` | `bookQuestionRepository.delete()` | Interviews page — remove a question from the book. Loads the BookQuestion first (to get its `bookId`), then verifies book ownership. Throws `NOT_FOUND` if bookQuestionId doesn't exist. |

- All procedures are protected. `getById`, `addQuestion`, and `removeQuestion` verify the book belongs to the authenticated user.
- `getById` throws `TRPCError({ code: 'NOT_FOUND' })` if the book doesn't exist (before the ownership check can even run).

### Interview Router (`src/server/routers/interview.ts`)

| Operation | Type | Input | Delegates to | UI Usage |
|-----------|------|-------|-------------|----------|
| `start` | mutation | `{ bookQuestionId: string }` | `conversationService.startInterview()` | Interviews page — "Start Interview" button. Verifies book ownership (bookQuestion → book → userId). Returns `{ interviewId, openingMessage }`, then client navigates to `/interview/[interviewId]`. |
| `getById` | query | `{ id: string }` | `interviewRepository.findById()` | Interview page — load interview metadata (status, questionId, bookId). Throws `NOT_FOUND` if interview doesn't exist. Used when resuming an interview. |
| `sendMessage` | mutation | `{ interviewId: string, content: string }` | `conversationService.sendMessage()` | Interview page — user sends a message, gets the AI response back. Throws `BAD_REQUEST` if interview status is `COMPLETE`. |
| `getMessages` | query | `{ interviewId: string }` | `conversationService.getInterviewMessages()` | Interview page — server-side load of full transcript on page load/refresh. |
| `complete` | mutation | `{ interviewId: string }` | `conversationService.completeInterview()` | Interview page — "Finish Interview" button. Marks interview as COMPLETE. Throws `BAD_REQUEST` if already `COMPLETE`. |

- All procedures are protected. All procedures verify the interview belongs to the authenticated user (interview → book → userId chain).
- `start` verifies ownership through the bookQuestion's book. Throws `NOT_FOUND` if bookQuestionId doesn't exist.

### Shared Helpers

- Create `src/server/routers/_helpers.ts`:
  - `verifyBookOwnership(bookId, userId)` — loads book, throws `NOT_FOUND` if missing, throws `FORBIDDEN` if `book.userId !== userId`, returns the book
  - `verifyInterviewOwnership(interviewId, userId)` — loads interview (throws `NOT_FOUND` if missing) → loads book → checks `book.userId === userId` (throws `FORBIDDEN`), returns the interview
  - `verifyBookQuestionOwnership(bookQuestionId, userId)` — loads bookQuestion (throws `NOT_FOUND` if missing) → delegates to `verifyBookOwnership(bookQuestion.bookId, userId)`, returns the bookQuestion
  - Used by book and interview routers to enforce access control before delegating to services

## Notes

- Per ADR 013, all conversation operations go through tRPC — no separate streaming API route. This gives us full type safety and a single API surface with no duplicated auth.
- The `interview.sendMessage` mutation awaits the full response from the conversation service and returns it. The client sees a loading state while waiting, then gets the complete response. For short interviewer responses this is snappy.
- Ownership verification is important: a user should never be able to access another user's interviews, books, or messages. The chain is always: resource → ... → book → check `book.userId === ctx.userId`.
- Middleware (`src/middleware.ts`) already allows `/api/*` as public routes (auth is enforced at the handler level), so no middleware changes needed.
- `question.create` has no authorization gating yet — any authenticated user can create questions. Proper admin authorization will be added later.
- `removeQuestion` takes a `bookQuestionId` (the join record), not a `bookId + questionId` pair. This is simpler and avoids a lookup.
- All helpers throw `NOT_FOUND` when a resource doesn't exist, before checking ownership. This prevents leaking information about other users' resources through different error codes.
- `interview.sendMessage` and `interview.complete` reject requests when the interview is already `COMPLETE` with `BAD_REQUEST`.
