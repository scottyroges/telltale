# Plan 0.2: Testing Infrastructure

**Status:** Complete
**Depends on:** 0.1 (need project structure and `package.json`)
**Blocks:** None (but all subsequent plans should include tests from the start)

## Goal

Install and configure Vitest, React Testing Library, and Playwright per ADR 010. Establish the hybrid test file organization (co-located for frontend, mirrored `tests/` for backend), create test helpers, and add a smoke test for each runner to prove the setup works.

## Manual Steps (Before Code)

None.

## Key Steps

### Vitest + React Testing Library

- Install `vitest`, `@vitejs/plugin-react`, `vite-tsconfig-paths`, `jsdom`
- Install `@testing-library/react`, `@testing-library/jest-dom`, `@testing-library/user-event`
- Install `@vitest/coverage-v8` for coverage reporting (uses V8's built-in instrumentation — fast, low overhead)
- Create `vitest.config.ts`:
  - `plugins: [react(), tsconfigPaths()]`
  - `test.environment: "jsdom"` for component tests
  - `test.include` covering both `src/**/*.test.{ts,tsx}` and `tests/**/*.test.{ts,tsx}`
  - `test.setupFiles` pointing to `src/test/setup.ts`
  - `test.coverage.provider: "v8"`
- Create `src/test/setup.ts` — imports `@testing-library/jest-dom/vitest`
- Create `tests/helpers/` directory for shared test utilities (factories, mocks) — add `.gitkeep` so Git tracks the empty directory
- Add `package.json` scripts:
  - `"test": "vitest run"`
  - `"test:watch": "vitest"`
  - `"test:coverage": "vitest run --coverage"`

### Playwright

- Install `@playwright/test`
- Run `npx playwright install --with-deps chromium` (Chromium only for dev, full matrix in CI)
- Create `playwright.config.ts`:
  - `testDir: "./e2e"`
  - `webServer` config to start Next.js on a test port
  - Projects: Chromium only for local dev, add Firefox + WebKit in CI
  - `baseURL: "http://localhost:3001"` (dedicated test port to avoid conflicts with a running dev server)
- Add `package.json` scripts:
  - `"test:e2e": "playwright test"`
  - `"test:e2e:ui": "playwright test --ui"`

### Smoke Tests

- Create `tests/smoke.test.ts` — trivial assertion to prove Vitest runs for backend tests
- Create `src/components/smoke-button.tsx` — minimal client component (`"use client"`, renders a button) solely to prove RTL works; will be replaced by a real component later
- Create `src/components/smoke-button.test.tsx` — render test for `SmokeButton` to prove RTL works with co-located frontend tests
- Create `e2e/smoke.spec.ts` — navigate to `/` and assert the page loads

### Configuration

- Add to `.gitignore`: `playwright-report/`, `test-results/`, `coverage/`
- Update `tsconfig.json` if needed to include `tests/` in compilation

## Notes

- Vitest config must resolve the `@/*` path alias via `vite-tsconfig-paths` — without this, imports break in test files
- `jsdom` environment is needed for component tests but not for pure logic tests — Vitest uses the `environment` option or per-file `@vitest-environment` comments
- Playwright `webServer` should use `npm run dev` for local and `npm run build && npm start` for CI — use port 3001 (`PORT=3001`) to avoid conflicts with a running dev server
- Only install Chromium locally — the full browser matrix runs in CI to keep local setup fast
- `SmokeButton` is a throwaway component — replace or delete it once a real client component exists to test
