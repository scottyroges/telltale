# Plan 1.2: Anthropic SDK + Conversation Service

**Status:** Planned
**Depends on:** 1.1 (needs repositories and domain types)
**Blocks:** 1.3, 1.5, 1.6

## Goal

Install the Anthropic SDK, create the client wrapper, and build the core conversation service with the initial interviewer system prompt. This is the heart of the product — the service that orchestrates conversations with Claude.

## Manual Steps (Before Code)

- `npm install @anthropic-ai/sdk`
- Ensure `ANTHROPIC_API_KEY` is set in `.env` (already in `.env.example`)

## Key Steps

- Create `src/lib/anthropic.ts`:
  - Singleton Anthropic client, same pattern as `src/lib/prisma.ts`
  - `import "server-only"` guard
  - Reads `ANTHROPIC_API_KEY` from environment
  - Exports `anthropic` client instance
- Create `src/services/prompt.ts`:
  - Exports `INTERVIEWER_SYSTEM_PROMPT` as a string constant
  - Warm, curious interviewer persona
  - Instructions: ask open-ended follow-ups, pick up on names/places/emotions, avoid yes/no questions, keep responses conversational (not too long), reference earlier details to show active listening
  - Includes role framing: "You are a skilled life story interviewer helping someone capture their personal history..."
- Create `src/services/conversation.service.ts`:
  - **`startSession(storyId: string)`**
    - Creates a new StorySession via `sessionRepository.create()`
    - Creates the opening ASSISTANT message: a warm greeting + initial open-ended question based on the story's topic
    - To generate the opening, calls Claude with the system prompt + a single user message like "The topic for this conversation is: {topic}. Please greet the storyteller warmly and ask an opening question about this topic."
    - Persists the assistant message via `messageRepository.create()`
    - Returns `{ sessionId, openingMessage }`
  - **`sendMessage(sessionId: string, content: string)`**
    - Persists the user message via `messageRepository.create()`
    - Loads message history via `messageRepository.findBySessionId()`
    - Builds the messages array: system prompt + message history (mapped to Anthropic format)
    - Calls `anthropic.messages.create()` and awaits the full response
    - Persists the assistant response via `messageRepository.create()`
    - Returns the assistant response text as a plain string
  - **`getSessionMessages(sessionId: string)`**
    - Returns messages via `messageRepository.findBySessionId()`
  - **`completeSession(sessionId: string)`**
    - Updates session status to COMPLETE via `sessionRepository.updateStatus()`
- Write unit tests in `tests/services/conversation.service.test.ts`:
  - Mock `@/lib/anthropic` (the SDK client)
  - Mock all repositories
  - Use `@vitest-environment node` directive
  - Test `startSession`: creates session, calls Claude for opening, persists message
  - Test `sendMessage`: persists user message, loads history, calls Claude with correct messages, returns response text, persists assistant response
  - Test `getSessionMessages`: delegates to repository
  - Test `completeSession`: updates status

## Notes

- The conversation service is **framework-agnostic** — no Next.js, tRPC, or React imports. It imports only from repositories, domain types, and lib.
- `sendMessage` returns a plain string, not a stream. Per ADR 013, the interviewer model produces short responses (a reflection + follow-up question) where streaming adds complexity without UX benefit. Streaming can be added later if lag becomes noticeable.
- The Anthropic SDK's `.create()` method returns the full message response. We extract the text content and return it directly.
- `startSession` returns the opening message synchronously — it's a short greeting.
- The system prompt is a separate file so it can be imported by both the conversation service (Plan 1.2) and the thread extraction service (Plan 1.5) if needed.
- Model selection: use `claude-sonnet-4-5-20250929` (Claude Sonnet 4.5). Can be made configurable later.
- Token usage tracking is not included in this plan — can be added later as a cross-cutting concern.
