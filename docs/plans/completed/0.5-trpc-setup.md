# Plan 0.5: tRPC Setup

**Status:** Complete
**Depends on:** 0.4 (need Better Auth providing session data for tRPC context)
**Blocks:** 0.6

## Goal

tRPC v11 with auth-aware context, health-check router (public + authenticated), server-side caller for Server Components, React Query client for Client Components. The dashboard proves the full stack works end-to-end. This plan also establishes the first service and repository to set the layering pattern for all future routers.

## Manual Steps (Before Code)

None.

## Key Steps

- Install `@trpc/server`, `@trpc/client`, `@trpc/tanstack-react-query`, `@tanstack/react-query@5`, `zod`, `superjson`
  - Note: `server-only` is already installed from Plan 0.4
- Create `src/server/trpc.ts` (replaces stub):
  - `createTRPCContext()` — calls `auth.api.getSession({ headers: await headers() })` to get session, returns `{ session, userId }`
  - `publicProcedure` — no auth required
  - `protectedProcedure` — requires session, narrows context to guarantee `userId`
  - `superjson` transformer via `initTRPC.create({ transformer: superjson })`
  - Export `createCallerFactory` from the `t` object for server-side callers
- Create `src/repositories/user.repository.ts`:
  - `findById(id: string)` — returns user by ID via Prisma
  - First repository in the project — establishes the pattern
- Create `src/services/health.service.ts`:
  - `checkDatabase(userId: string)` — calls `UserRepository.findById()`, returns user info + timestamp
  - First service in the project — establishes the pattern
- Create `src/server/routers/health.ts`:
  - `ping` — public, returns `{ status: "ok", timestamp }`
  - `dbCheck` — protected, delegates to `HealthService.checkDatabase()`, returns user info + timestamp
- Create `src/server/routers/_app.ts` — root router, exports `AppRouter` type
- Create `src/app/api/trpc/[trpc]/route.ts` — fetch adapter handler (`GET` + `POST`)
- Update `src/middleware.ts` — add `/api/trpc` to public routes allowlist (tRPC enforces auth via `protectedProcedure`, not middleware)
- Create `src/lib/trpc/server.ts`:
  - `serverTRPC()` — uses `t.createCallerFactory(appRouter)` to create caller from context
  - Import guarded with `"server-only"`
- Create `src/lib/trpc/client.tsx` — React tRPC client with `httpBatchLink` + `superjson` transformer (must match server)
- Create `src/components/providers.tsx` — single provider file composing `QueryClientProvider` + `TRPCProvider` (extensible for future theme/toast providers)
- Update `src/app/layout.tsx` — wrap children with `Providers`
- Update `src/app/(app)/dashboard/page.tsx`:
  - Call `health.dbCheck` via `serverTRPC()`
  - Display user name, ID, DB connection status, timestamp
- Create `tests/server/routers/health.test.ts`:
  - Test `ping` returns status and timestamp
  - Test `dbCheck` returns user info when authenticated
  - Test `dbCheck` throws UNAUTHORIZED when no session
  - Uses `createCallerFactory` to call procedures directly (no HTTP)

## Notes

- tRPC v11 uses `@trpc/tanstack-react-query` (not the old `@trpc/react-query`)
- tRPC v11 uses `t.createCallerFactory(appRouter)` — not `appRouter.createCaller()` directly
- `serverTRPC` pattern avoids HTTP round-trip in Server Components — calls procedures directly
- `superjson` transformer must be configured in **both** `initTRPC.create()` (server) and `httpBatchLink` (client) — mismatch causes Date serialization failures
- Middleware allows `/api/trpc/*` through as public; auth enforcement happens at the tRPC procedure level via `protectedProcedure`
- Using simple `createCallerFactory` for server-side caller (not `createTRPCOptionsProxy` + hydration). Can upgrade to the prefetch-and-hydrate pattern later when client components need server-fetched data in React Query cache.
- `providers.tsx` is a single composer file — when future providers are needed (theme, toast), add them here
