# Plan 0.5: tRPC Setup

**Status:** Not Started
**Depends on:** 0.4 (need Auth.js providing session data for tRPC context)
**Blocks:** 0.6

## Goal

tRPC v11 with auth-aware context, health-check router (public + authenticated), server-side caller for Server Components, React Query client for Client Components. The dashboard proves the full stack works end-to-end.

## Manual Steps (Before Code)

None.

## Key Steps

- Install `@trpc/server`, `@trpc/client`, `@trpc/tanstack-react-query`, `@tanstack/react-query`, `zod`, `superjson`, `server-only`
- Create `src/server/trpc.ts` (replaces stub):
  - `createTRPCContext()` — calls `auth()` to get session, returns `{ session, userId }`
  - `publicProcedure` — no auth required
  - `protectedProcedure` — requires session, narrows context to guarantee `userId`
  - `superjson` transformer for Date serialization
- Create `src/server/routers/health.ts`:
  - `ping` — public, returns `{ status: "ok", timestamp }`
  - `dbCheck` — protected, queries user from DB via Prisma, returns user info + timestamp
- Create `src/server/routers/_app.ts` — root router, exports `AppRouter` type
- Create `src/app/api/trpc/[trpc]/route.ts` — fetch adapter handler (`GET` + `POST`)
- Create `src/lib/trpc/server.ts`:
  - `serverTRPC()` — creates caller from `appRouter.createCaller(ctx)`
  - Import guarded with `"server-only"`
- Create `src/lib/trpc/client.tsx` — React tRPC client with `httpBatchLink` + `superjson`
- Create `src/components/trpc-provider.tsx` — `QueryClientProvider` + `TRPCProvider`
- Update `src/components/providers.tsx` — add `TRPCReactProvider`
- Update `src/app/(app)/dashboard/page.tsx`:
  - Call `health.dbCheck` via `serverTRPC()`
  - Display user name, ID, DB connection status, timestamp

## Notes

- tRPC v11 uses `@trpc/tanstack-react-query` (not the old `@trpc/react-query`)
- `serverTRPC` pattern avoids HTTP round-trip in Server Components — calls procedures directly
- `superjson` is critical for Date serialization across the wire
