# Code Reviewer Memory

## Project Patterns
- Repositories use `columns` array at module level to select domain type fields; `as const` for type safety
- Repository tests: `// @vitest-environment node`, mock `server-only`, mock `@/lib/db` with chainable proxy (via `tests/helpers/mock-db.ts`)
- Domain types are plain TS types in `src/domain/`, re-exported from `src/domain/types.ts`
- `user.repository.ts` does NOT use domain types (returns inferred Kysely types) -- newer repos do
- Application models use `@default(cuid())` for IDs; Better Auth models do not
- `Message` and `InterviewSummary` have no `updatedAt` (append-only)
- Plan files live at `docs/plans/active/` (or `completed/`) and serve as the spec
- Query layer: Prisma for schema/migrations, Kysely for queries (ADR 015)
- `@prisma/client` stays in deps for Better Auth; repos import from `@/lib/db` (Kysely)
- ID generation: `@paralleldrive/cuid2` via `src/lib/id.ts`
- `src/db/types.ts` and `src/db/enums.ts` are generated by `prisma-kysely`

## Review Learnings
- Check "Done When" checklist items literally -- e.g. `data-model.md` update requirement
- `InterviewSummary.parentSummaryId` has `@unique` constraint for 1:1 linked list
- `jsonArrayFrom`/`fn.toJson` in Kysely: dates inside JSON aggregations come back as ISO strings, NOT Date objects -- `pg` driver only parses dates for top-level columns, not JSON-embedded values
- ADR 015 has a stale bullet about removing `@prisma/client` from runtime deps -- it stays for Better Auth
