# Plan 1.4: Chat UI

**Status:** Planned
**Depends on:** 1.3 (needs tRPC routers)
**Blocks:** 1.5, 1.6

## Goal

Build the minimal chat interface — message display, text input, and response rendering. Completes the first end-to-end loop: a logged-in user can have a multi-turn conversation with Claude and see the full transcript persist across page refreshes.

## Manual Steps (Before Code)

None.

## Key Steps

- Create `src/app/(app)/stories/page.tsx`:
  - Server Component — lists the user's stories via `serverTRPC().story.list()`
  - Shows each story with topic, status, session count
  - "New Story" button/form — input for topic, calls `trpc.story.create` mutation
  - Each story links to its latest session or has a "Start Conversation" button
  - "Start Conversation" calls `trpc.session.start`, then navigates to `/conversation/[sessionId]`
  - Functional styling only (CSS Modules) — not polished
- Create conversation page and components:
  - `src/app/(app)/conversation/[sessionId]/page.tsx`:
    - Server Component that loads initial messages via `serverTRPC().session.getMessages({ sessionId })`
    - Passes messages as props to the client-side Chat component
    - Verifies the session exists and belongs to the user (404 if not)
  - `src/components/conversation/chat.tsx`:
    - Client Component — the main chat orchestrator
    - Receives initial messages as props
    - Uses `trpc.session.sendMessage` mutation for sending messages
    - Manages local message list: initial messages + optimistic user messages + response
    - Shows a loading/thinking indicator while waiting for the AI response
    - After response arrives, adds it to the message list and invalidates the tRPC query to sync with server state
  - `src/components/conversation/message-list.tsx`:
    - Renders a scrollable list of messages
    - Auto-scrolls to bottom on new messages
    - Shows a "thinking" indicator while awaiting the AI response
  - `src/components/conversation/message-bubble.tsx`:
    - Renders a single message with role-based styling (user messages right-aligned, assistant messages left-aligned)
    - Shows timestamp
  - `src/components/conversation/message-input.tsx`:
    - Text input with send button
    - Submit on Enter (Shift+Enter for newline)
    - Disabled while awaiting response
    - Auto-focus on mount
- Update `src/app/(app)/dashboard/page.tsx`:
  - Add a link/button to navigate to `/stories`
- CSS Modules for all new components:
  - `src/components/conversation/chat.module.css`
  - `src/components/conversation/message-list.module.css`
  - `src/components/conversation/message-bubble.module.css`
  - `src/components/conversation/message-input.module.css`
  - `src/app/(app)/stories/stories.module.css`
  - Functional layout — nothing fancy, just readable and usable

## Notes

- The stories page handles the "new conversation" flow: create a story topic → start a session → redirect to conversation page. This is the simplest possible UX for MVP.
- The conversation page is a Server Component that loads initial data, with a Client Component (`Chat`) handling the interactive parts. This follows the Next.js pattern of pushing client boundaries down.
- No custom streaming hook needed — the Chat component uses a standard tRPC mutation (`trpc.session.sendMessage`). React Query handles loading/error states automatically. Per ADR 013, short interviewer responses don't benefit from token-by-token streaming.
- After a response arrives, the Chat component should invalidate/refetch the `session.getMessages` query so the server-persisted messages are the source of truth. The optimistic messages are replaced by the real ones.
- Auto-scroll behavior: scroll to bottom when new messages arrive, but only if the user is already at/near the bottom (don't force-scroll if they've scrolled up to read earlier messages).
- The conversation page uses a dynamic route `[sessionId]` — not `[storyId]`. This is because a story can have multiple sessions, and we want to show a specific session's conversation.
- CSS Modules keep styles scoped and avoid conflicts. No global styles or CSS-in-JS.
- Stories page uses a Client Component wrapper for the "New Story" form that calls tRPC mutations. The page itself is a Server Component.
