# Plan 1.4: Interview UI

**Status:** Complete
**Depends on:** 1.3 (needs tRPC routers)
**Blocks:** 1.5, 1.6

## Goal

Build the interview interface — the core experience where a user has a guided conversation with an AI interviewer. Completes the first end-to-end loop: a logged-in user can browse questions, start an interview, have a multi-turn conversation, and see the full transcript persist across page refreshes.

This is an **interview**, not a chatbot. The UI should feel like sitting down with a thoughtful interviewer — focused, warm, unhurried. The interviewer guides the conversation; the user tells their story.

**Content & behavior:**

- Page loads via `serverTRPC` using `trpc.book.getById({ id: bookId })` and `trpc.question.list()`
- Back link navigates to `/books`
- Book title displayed as page heading

- **"Your Questions" section** — lists every BookQuestion in the book (from `BookWithDetails.bookQuestions`)
  - Each row shows the question prompt text and a status indicator (based on `BookQuestion.status`):
    - `NOT_STARTED` — gray dot, "Not Started", **"Begin"** button
    - `STARTED` — blue dot, "In Progress", **"Continue"** link
    - `COMPLETE` — green checkmark, "Complete", **"Review"** link (read-only transcript)
  - "Begin" calls `trpc.interview.start({ bookQuestionId })`, then navigates to `/interview/[interviewId]`
  - "Continue" finds the matching interview from `BookWithDetails.interviews` (matched by `questionId`) and links to `/interview/[interviewId]`
  - "Review" links to `/interview/[interviewId]` (same page, but input is hidden for completed interviews)
  - If the book has no questions yet, show an empty state: "You haven't added any questions yet. Browse the catalog below to get started."

- **"Add More Questions" section** — browse the global question catalog
  - Category dropdown filter (defaults to "All") — categories derived client-side from `question.list()` results (**tech debt:** add a `question.listCategories` procedure when the catalog grows)
  - List of questions not already in the book (filter `question.list()` results against `BookWithDetails.bookQuestions`)
  - Each row shows the question prompt text and an **"Add"** button
  - "Add" calls `trpc.book.addQuestion({ bookId, questionId })` and moves the question into "Your Questions" above
  - Questions already in the book don't appear here

**States:**
- **Loading:** Skeleton placeholders for both sections
- **Empty book:** "Your Questions" section shows the empty state message
- **Empty catalog:** "Add More Questions" section shows "No more questions available"

### Page 4: Interview Session (`/interview/[interviewId]`)

The heart of the app. A focused conversation between the user and the AI interviewer.

**Layout:**

```
┌─────────────────────────────────────────────────┐
│  [← Back]    "What was your childhood like?"    │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌───────────────────────────────────┐          │
│  │ Hi! I'd love to hear about your   │          │
│  │ childhood. What's the first thing  │          │
│  │ that comes to mind when you think  │          │
│  │ about growing up?                  │          │
│  └───────────────────────────────────┘          │
│                                                 │
│          ┌───────────────────────────────┐      │
│          │ We lived in this small town    │      │
│          │ in Ohio. My dad worked at the  │      │
│          │ factory and my mom stayed home │      │
│          │ with us kids.                  │      │
│          └───────────────────────────────┘      │
│                                                 │
│  ┌───────────────────────────────────┐          │
│  │ That sounds like a close-knit      │          │
│  │ family. What was it like having    │          │
│  │ your mom home? Do you remember     │          │
│  │ any routines or traditions?        │          │
│  └───────────────────────────────────┘          │
│                                                 │
│          ┌───────────────────────────────┐      │
│          │ ...                            │      │
│          └───────────────────────────────┘      │
│                                                 │
├─────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────┐  │
│  │ Type your response...                     │  │
│  │                                     [Send]│  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
```

**Data loading:** Server Component loads via `serverTRPC`:
- `interview.getById({ id: interviewId })` — gets interview metadata (bookId, questionId, status)
- `interview.getMessages({ interviewId })` — gets the message transcript
- The question prompt text for the header is derived from the interview's associated question (loaded via the book's `bookQuestions`)

**Top bar:**
- Back arrow/link → navigates to `/book/[bookId]/interviews` (bookId from the interview object)
- Question prompt text as the page title (the topic being discussed)

**Transcript area** (scrollable, fills available vertical space):
- Messages displayed in conversation order
- **Skip the first message** — the `startInterview` service stores an initial topic prompt as a USER-role message (e.g., "The topic for this conversation is: ..."). This is an internal system message, not something the user wrote. Filter it out so the transcript begins with the interviewer's opening ASSISTANT message.
- **Interviewer messages** (ASSISTANT role): left-aligned, subtle background (e.g., light gray)
- **User messages** (USER role): right-aligned, distinct background (e.g., light blue)
- No avatars, no timestamps for MVP — keep it clean and focused on the words
- **Thinking indicator**: when waiting for the AI response, show a subtle "..." animation below the last message, left-aligned (where the interviewer's next message will appear)
- Auto-scroll to bottom on new messages, but only if user is already near the bottom (don't interrupt if they scrolled up to re-read)

**Input area** (fixed to bottom of viewport):
- Auto-expanding textarea (grows with content, up to a max height)
- Send button to the right of the textarea
- Submit on Enter, Shift+Enter for newline
- Disabled while waiting for AI response (prevent double-send)
- Auto-focus on mount so user can start typing immediately
- For completed interviews (`InterviewStatus.COMPLETE`): input area is hidden, transcript is read-only

**States:**
- **Loading:** Transcript area shows skeleton, input area hidden until loaded
- **Sending:** User message appears immediately (optimistic), input disabled, thinking indicator shows
- **Error:** If `sendMessage` fails, show inline error below the input ("Something went wrong. Try again.") with a retry option
- **Empty (just started):** Only the interviewer's opening message is shown

## Notes

- **Interview, not chat.** The language everywhere — component names, CSS classes, variable names — should reflect "interview" and "transcript", not "chat" and "messages". The user is being interviewed; they're not messaging a bot.
- **Server Components load data, Client Components handle interaction.** The pages are Server Components that fetch initial data via `serverTRPC()`. Interactive parts (sending messages, starting interviews, adding questions) are pushed into Client Components.
- **No streaming.** Per ADR 013, interviewer responses are short and don't benefit from token-by-token streaming. The `sendMessage` mutation returns the full response. React Query handles loading/error states.
- **Optimistic updates for messages.** When the user sends a message, it appears immediately in the transcript (optimistic). After the AI responds, the component invalidates/refetches `interview.getMessages` so server-persisted messages become the source of truth.
- **Three component directories.** `src/components/interview/` holds the session/transcript components. `src/components/book-interviews/` holds the question list and catalog components. `src/components/books/` holds the book list. This mirrors the route structure.
- **CSS Modules for all styling.** Scoped styles, no global CSS or CSS-in-JS. Functional layout only — readable and usable, not polished.
- **Completed interviews are read-only.** Same interview page, but the input area is hidden. The transcript is still viewable.
- **Two status enums.** `BookQuestionStatus` (`NOT_STARTED | STARTED | COMPLETE`) drives the status indicators on the book interviews page. `InterviewStatus` (`ACTIVE | PAUSED | COMPLETE`) is the interview's own lifecycle status — used to determine whether the input area is shown on the interview page. Don't confuse them.
