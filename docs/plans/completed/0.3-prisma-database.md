# Plan 0.3: Prisma + Database

**Status:** Complete
**Depends on:** 0.1 (need `package.json` and project structure)
**Blocks:** 0.4, 0.5, 0.6

## Goal

Prisma ORM with Auth.js models (User, Account, Session, VerificationToken), Neon PostgreSQL connection, initial migration, client singleton, and seed script. Schema is minimal — no Book/Story/etc yet.

## Manual Steps (Before Code)

### Neon (Production)
1. **Create Neon account** at https://neon.tech (free tier)
2. **Create project** named "telltale" — choose region closest to your users
3. **Copy connection strings:**
   - Pooled connection (has `-pooler` in hostname) → `DATABASE_URL`
   - Direct connection (no `-pooler`) → `DIRECT_URL`

### Local Development
4. **Docker required** — local Postgres via `docker-compose.yml`
5. **Run** `docker compose up -d` to start local Postgres
6. **Create `.env`** from `.env.example` — local defaults pre-filled, swap for Neon strings in production

## Key Steps

- Install `prisma`, `@prisma/client`, `tsx`
- `npx prisma init --datasource-provider postgresql`
- Write `prisma/schema.prisma`:
  - Datasource with `url` (pooled) and `directUrl` (direct)
  - User, Account, Session, VerificationToken models per Auth.js Prisma adapter spec
  - `createdAt`/`updatedAt` on User, Account, Session
- Create `src/lib/prisma.ts` — plain PrismaClient singleton with `globalThis` caching for dev
- Create `prisma/seed.ts` — upserts a dev user (`dev@telltale.local`)
- Add to `package.json`:
  - `"postinstall": "prisma generate"`
  - `"db:migrate": "prisma migrate dev"`
  - `"db:push": "prisma db push"`
  - `"db:seed": "prisma db seed"`
  - `"db:studio": "prisma studio"`
  - `"prisma": { "seed": "tsx prisma/seed.ts" }`
- Run `npx prisma migrate dev --name init`
- Update `.env.example` to include `DIRECT_URL`

## Decisions (from pre-implementation review)

- **Plain Prisma over Neon adapter.** Dropped `@prisma/adapter-neon` and `@neondatabase/serverless`. Standard PostgreSQL connection is simpler and sufficient — edge runtime support can be added later if needed.
- **No `previewFeatures` needed.** `driverAdapters` was only relevant for the Neon adapter.
- **`tsx` for seed script.** Lightweight, purpose-built for running TypeScript directly.
- **Local Postgres via Docker for dev.** `docker-compose.yml` with a Postgres container. `.env.example` ships with localhost defaults so dev works out of the box after `docker compose up -d`. Neon is for production only.
- **Verify `.gitignore` covers `.env`** before adding real credentials.
- **Repository layer deferred.** `src/lib/prisma.ts` is internal plumbing; the repository abstraction (`src/repositories/`) comes later when the first feature needs it.
