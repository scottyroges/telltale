# Plan 0.4: Better Auth Configuration

**Status:** Not Started
**Depends on:** 0.3 (need Prisma schema and working database)
**Blocks:** 0.5, 0.6

## Goal

Better Auth fully configured with Prisma adapter, Google OAuth, sign-in/sign-out pages, middleware protecting the `(app)` route group, and session access server-side via `auth.api.getSession()`.

## Manual Steps (Before Code)

1. **Create Google OAuth credentials:**
   - Go to https://console.cloud.google.com
   - Create project "Telltale" (or use existing)
   - APIs & Services → Credentials → Create OAuth client ID
   - Type: Web application
   - Authorized JS origins: `http://localhost:3000`
   - Authorized redirect URIs: `http://localhost:3000/api/auth/callback/google`
   - Copy Client ID and Client Secret

2. **Generate BETTER_AUTH_SECRET:**
   - Run `openssl rand -base64 33`

3. **Update `.env`:**
   - `BETTER_AUTH_SECRET="<generated>"`
   - `BETTER_AUTH_URL="http://localhost:3000"`
   - `GOOGLE_CLIENT_ID="<client-id>"`
   - `GOOGLE_CLIENT_SECRET="<client-secret>"`

## Key Steps

- Install `better-auth`
- Generate Better Auth schema: `npx @better-auth/cli generate` — outputs Prisma models, merge into `prisma/schema.prisma`
- Reset DB and run a fresh migration (`npx prisma migrate dev --name better-auth`)
- Create `src/server/auth.ts`:
  - `betterAuth()` with `prismaAdapter(prisma, { provider: "postgresql" })`
  - `socialProviders: { google: { clientId, clientSecret } }`
  - Export `auth` instance
- Create `src/app/api/auth/[...all]/route.ts` — `toNextJsHandler(auth)` exporting `GET`/`POST`
- Create `src/lib/auth-client.ts` — `createAuthClient()` from `better-auth/react`
- Create `src/middleware.ts`:
  - Use `betterFetch("/api/auth/get-session", ...)` pattern (Edge Runtime compatible)
  - Allow public routes: `/`, `/api/auth/*`
  - Redirect unauthenticated users to `/login`
  - Redirect authenticated users away from `/login` to `/dashboard`
  - Export `config.matcher` excluding `_next/static`, `_next/image`, `favicon.ico`
- Create `src/app/(auth)/layout.tsx` — minimal auth layout
- Create `src/app/(auth)/login/page.tsx` — Server Component shell
- Create `src/app/(auth)/login/google-sign-in-button.tsx` — Client Component using `authClient.signIn.social({ provider: "google" })`
- Create `src/app/(app)/layout.tsx` — calls `auth.api.getSession({ headers: await headers() })` server-side, renders header with user name + sign-out button
- Create `src/app/(app)/sign-out-button.tsx` — Client Component using `authClient.signOut()`
- Create `src/app/(app)/dashboard/page.tsx` — placeholder showing welcome message
- Root layout (`src/app/layout.tsx`) — no SessionProvider needed (Better Auth doesn't require one)

**Unit tests (`tests/middleware.test.ts`):**

- Authenticated request to `/login` redirects to `/dashboard`

**E2E tests (`e2e/auth.spec.ts`):**

- E2E smoke test: `/login` page renders with Google sign-in button

## Notes

- Email provider omitted for Phase 0 — Google OAuth is sufficient to prove auth works
- Better Auth uses `BETTER_AUTH_` env var prefix for its own config; Google credentials use standard `GOOGLE_CLIENT_ID`/`GOOGLE_CLIENT_SECRET`
- Better Auth uses DB sessions with signed cookie caching — no JWT strategy needed
- No type augmentation needed — Better Auth uses `$Infer` for type inference
- No SessionProvider needed — `authClient` from `better-auth/react` works without a provider
- Sign-in and sign-out buttons are Client Components (use `authClient.signIn.social()` and `authClient.signOut()`)
